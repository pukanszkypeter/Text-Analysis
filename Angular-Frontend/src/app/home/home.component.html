In the context of this application, I would like to present a small part of the possibilities provided by text analysis. The application will briefly talk about Python and SAP HANA environments, business opportunities and data visualization.

What is Text Analysis?

Text analysis is a machine learning technique that allows us to extract specific information from a text, such as keywords, names, or company information. We can even use it to categorize different emails or comments by topic or sentiment.

This method provides qualitative results, so it identifies important information from the text itself. Take, for example, the database that the application uses. We can categorize comments received for podcast channels based on words with positive or negative emotion. Basically, the challenge in text analysis is decoding the ambiguity of human language.

"If you torture the data long enough, it will confess to anything." - Ronald H. Coase

Why Text Analysis?

Nowadays, words like data science, big data and AI are becoming more and more popular and trendy. That’s why it’s worth getting to know and learn a little more about such technologies. Text analysis is an extremely spectacular and playful way to learn about these.

  • It has a very wide range of possibilities in case of analyzing whole documents, single sentences or even sub-sentences.
  • It is scalable, because these tools allows us to structure vast quantities of information in seconds and redirect extra resources to more important tasks.
  • It happens in real-time and with perfectly parameterized models we can detect urgent issues from customer feedbacks and take action rather sooner then later.
  • It delivers consistent criteria because they work more accurately than humans ever could.

Methods & Techniques

There are basic and more advanced text analysis techniques, each used for different purposes. (See the list down below.) In this case we will focus on the text classification, more precisely on sentiment and topic analysis part.

  • Word Frequency
  • Collocation
  • Concordance
  • Text Classification (Topic Analysis, Sentiment Analysis, Intent Detection)
  • Text Extraction (Keyword Extraction, Named Entity Recognition)
  • Word Sense Disambiguation
  • Clustering

Topic Analysis

It can automatically organize text by subject or theme. Take the application database, for example. Podcast channels can be categorized, so we can already tag incoming comments based on those categories.

Let's see the comment: "Very useful podcast, I can only recommend. I learned a lot about cryptocurrencies and bitcoin mining."

The expected categories would probably be: business, finance, technology, fintech

Sentiment Analysis

It uses powerful machine learning algorithms to automatically read and classify for opinion polarity (positive, negative, neutral) and beyond, into the feelings and emotions of the writer, even context and sarcasm.

For example the input: "I love this podcast, great quality, nice people, many laughs."

And the expected output: positive

How does it work?

Although there are quite a few explanations for understanding AI and data science processes, we have chosen two interesting paths for interpretation. First, let’s look at the OSEMN model, which is extremely similar to cooking processes. (Try replacing the word data with food.)

  • Obtain
    • This is the phase when we collect the data we want to analyze from some source.
  • Scrub
    • Here we prepare the data for further processes. We clean them of missing data, false values, bad formats, etc...
  • Explore
    • In this case, we interpret the data set to be processed, create visualizations and statistics.
  • Model
    • This is the main phase, the part when we produce the model whether clustering, classification or regression, etc. by.
  • iNterpret
    • The last part is when we draw conclusions from the data and evaluate and communicate the results.

Another approach might be this quote:

"A computer program is said to learn to perform a task T from experience E". - Tom Mitchell

In other words, if we want text analysis software to perform desired tasks, we need to teach machine learning algorithms how to analyze, understand and derive meaning from text. It's very similar to the way humans learn how to differentiate between topics, objects, and emotions.

Usecases
